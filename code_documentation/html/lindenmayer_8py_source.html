12 class lindenmayer(object):
13  '''
14  Lindenmayer system, also commonly known as L-System, is developed
15  by Aristid Lindenmayer in 1968 (reference: Mathematical models for
16  cellular interaction in development. Journal of Theoretical
17  Biology 18:280-315). It is a set of formal grammar of production
18  rules for rewiting an initial axiom or seed text over generations.
19 
20  This implementation defines 3 types of rules, also known as production
21  rules or predicates; replacement, probability, and function rules. Each
22  rule can be given a priority.
23 
24  The simplest form of production rule takes the form of 'A -> BAC', which
25  is read as "whenever 'A' is found, it is replaced/rewritten as 'BAC'".
26  For example, if the starting axiom is "A", then the following will happen
27 
28  - Generation 0: A
29  - Generation 1: BAC
30  - Generation 2: BBACC
31  - Generation 3: BBBACCC
32  - Generation 4: BBBBACCCC
33  - and so on.
34 
35  In this case, the predicate 'A -> BAC' can be written as in 4 different
36  ways - C{['A', 'BAC']}, C{['A', 'BAC', 1]}, C{['A', 'BAC', 1,
37  'replacement']}, or C{['A', 'BAC', 1, 'replacement', 1]}.
38 
39  When a list of 2-elements is given (e.g., C{['A', 'BAC']}), it is taken
40  to be replacement rule with the highest priority; that is, priority of 1.
41  Hence, C{['A', 'BAC']}, C{['A', 'BAC', 1]}, C{['A', 'BAC', 1,
42  'replacement']}, and C{['A', 'BAC', 1, 'replacement', 1]} are the same.
43 
44  This also means that production rules can have different priorities. For
45  example, given C{[['A', 'BAC', 1], ['B', 'BC', 2]]}, rule C{['A', 'BAC',
46  1]} will be executed before C{['B', 'BC', 2]} in the following manner
47 
48  - Generation 0: A
49  - Generation 1: BCAC # A -> BAC, BAC -> BCAC
50  - Generation 2: BCCBCACC
51  - Generation 3: BCCCBCCBCACCC
52  - and so on as all production rules in ascending order of priorities
53  (with '1' being the highest priority) will be executed in sequence on
54  the resulting axiom at that current point in time. As a result, the
55  order of rules within the same priority is significant if they
56  recognize the same instruction or command.
57 
58  However, if given C{[['A', 'BAC', 1], ['B', 'BC', 1]]}, then
59 
60  - Generation 0: A
61  - Generation 1: BAC
62  - Generation 2: BCBACC
63  - Generation 3: BCCBCBACCC
64  - Generation 4: BCCCBCCBCBACCCC
65  - and so on.
66 
67  The second form of production rule is probabilistic, also known as
68  stochastic grammars. Probabilistic rule will take the format of
69  C{[<domain>, <range>, <priority>, 'probability', <probability>]}.
70  For example, C{['A', 'BAC', 1, 'probability', 0.5]} means that 'A'
71  will only be rewritten into 'BAC' 50% of the time. 'A' will be left
72  unchanged 50% of the time. The same priority principle applies.
73  Hence, C{['A', 'BAC', 1, 'probability', 1]} is in effect the same as
74  C{['A', 'BAC', 1, 'replacement']}.
75 
76  The third form of production rule is function rule, which takes the
77  form of C{[<domain>, <function>, <priority>, 'function']}. For
78  example, C{['A', axiom_func, 1, 'function']} means that when 'A' is
79  encounted in the axiom, the command string up to that point in time
80  will be used as parameter for axion_func function, such as
81 
82  - Generation 0: A
83  - Generation 1: dependent on the return value of axiom_func('A')
84  and so on.
85 
86  For example, given an axiom of 'ACCCABABDD', and C{['AB', replaceFunction,
87  1, 'function']} as production rule where replaceFunction is defined as
88 
89  >>> def replaceFunction(instructions, position):
90  >>> if instructions[position+3] == 'O': return 'BAAB'
91  >>> elif instructions[position-1] == 'O': return 'AABB'
92  >>> else: return 'OOAB'
93 
94  - Generation 0: ACCCABABDD
95  - Generation 1: ACCCOOABOOABDD
96  - Generation 2: ACCCOOBAABOOAABBDD
97  - Generation 3: ACCCOOBABAABOOAABBDD
98  - Generation 4: ACCCOOBABABAABOOAABBDD
99  - Generation 5: ACCCOOBABABABAABOOAABBDD
100 
101  In summary, the following rule formats are allowed:
102  - C{[<domain>, <range>]}
103  - C{[<domain>, <range>, <priority>]}
104  - C{[<domain>, <range>, <priority>, 'replacement']}
105  - C{[<domain>, <range>, <priority>, 'probability', <probability>]}
106  - C{[<domain>, <function>, <priority>, 'function']}
107  '''

119  def add_rules(self, rules):
120  '''
121  Method to add a list of production rules / predicates into the
122  system.
123 
124  @param rules: a list of list describing the production rules. Please
125  see above for rule syntax.
126  @type rules: list
127  '''
128  for x in rules:
129  if len(x) == 2:
130  # [predicate, replacement]
131  self.rules.append([x[0], x[1], 1, 'replacement', None])
132  elif len(x) == 3:
133  # [predicate, replacement, priority]
134  self.rules.append([x[0], x[1], int(x[2]), 'replacement', None])
135  elif len(x) == 4 and (x[3] not in ['probability',
136  'replacement',
137  'function']):
138  # [predicate, replacement, priority, '<something_else>']
139  print('''Warning: Rule type can only be 'probabilistic', \
140  'replacement' or 'function'. Rule, %s, is not added into \
141  system.''' % str(x))
142  elif len(x) == 4 and x[3] == 'replacement':
143  # [predicate, replacement, priority, 'replacement']
144  self.rules.append([x[0], x[1], int(x[2]), x[3], None])
145  elif len(x) == 4 and x[3] == 'probability':
146  # [predicate, replacement, priority, 'probability']
147  print('''Warning: Function rule will require a \
148  probability. Rule, %s, is added into system as a \
149  replacement rule (100% activation probability).''' % str(x))
150  self.rules.append([x[0], x[1], int(x[2]), 'replacement', None])
151  elif len(x) == 5 and x[3] == 'probability':
152  # [predicate, replacement, priority, 'probability',
153  # probability]
154  self.rules.append([x[0], x[1], int(x[2]), x[3], float(x[4])])
155  elif len(x) == 4 and x[3] == 'function':
156  # [predicate, function, priority, 'function']
157  self.rules.append([x[0], x[1], int(x[2]), x[3], None])
158  self.priority_levels = [x[2] for x in self.rules][-1]

160  def _apply_priority_rules(self, priority, data_string):
161  '''
162  Private method - to be used by apply_rules method to apply production
163  rules of a particular priority.
164 
165  @param priority: order of priority
166  @type priority: integer
167  @param data_string: data or symbol string to be processed
168  @type data_string: string
169  @return: rewritten data_string
170  '''
171  rules = [x for x in self.rules if x[2] == int(priority)]
172  ndata = ''
173  pointer = 0
174  while pointer < len(data_string):
175  cmd = data_string[pointer:pointer+self.command_length]
176  for rule in rules:
177  if cmd == rule[0] and rule[3] == 'replacement':
178  cmd = rule[1]
179  break
180  if cmd == rule[0] and rule[3] == 'probability' \
181  and random.random() < x[4]:
182  cmd = rule[1]
183  break
184  if cmd == rule[0] and rule[3] == 'function':
185  cmd = rule[1](data_string, pointer)
186  break
187  if cmd == None: cmd = ''
188  ndata = ndata + cmd
189  pointer = pointer + self.command_length
190  return ndata

192  def _apply_rules(self, data_string):
193  '''
194  Private method - to apply all production rules on axiom string (in
195  the first generation) or data/symbol string (in the subsequent
196  generations).
197  This method is implemented as a generator.
198 
199  @param data_string: data or symbol string to be processed
200  @type data_string: string
201  @return: rewritten data_string
202  '''
203  for priority in list(range(1, self.priority_levels+1)):
204  data_string = self._apply_priority_rules(priority,
205  data_string)
206  return data_string

208  def generate(self, axiom, iterations):
209  '''
210  Method to apply all production rules on an initial axiom string
211  over a number of iterations.
212 
213  @param axiom: data or symbol string to be processed
214  @type axiom: string
215  @param iterations: number of repetitions / iterations
216  @type iterations: integer
217  @return: rewritten axiom
218  '''
219  self.axiom = axiom
220  iterations = int(iterations)
221  count = 1
222  while count < iterations + 1:
223  self.axiom = self._apply_rules(self.axiom)
224  print('Generation %s: Axiom length = %s' % (str(count),
225  str(len(self.axiom))))
226  count = count + 1
227  return self.axiom

236

239 
240  A mapping dictionary is used to convert the symbol string into
241  Turtle commands. The following Turtle commands are defined:
242  forward, backward, right (turn), left (turn), pen up, pen down
243  and home. The default mapping is given as
244

246  >>> 'random_angle': 0,
247  >>> 'set_distance': 1,
248  >>> 'random_distance': 0,
249  >>> 'set_heading': 0,

251  >>> 'background_colour': 'ivory',
252  >>> 'F': 'forward',
253  >>> 'B': 'backward',
254  >>> 'R': 'right',
255  >>> 'L': 'left',
256  >>> 'H': 'home',
257  >>> 'U': 'penup',
258  >>> 'D': 'pendown',
259  >>> '[': 'push',
260  >>> ']': 'pop'}
261 
262  which can be read as
263  - a left or right turn is set at 90 degrees (set_angle). B{This
264  setting is mandatory.}
265  - random angles of turn can be set using 'random_angle', where
266  the actual angle will be from the set_angle to set_angle +
267  random_angle (by uniform distribution). For example, if
268  random_angle is 10 degrees, it means that the actual angle
269  at each turn will be uniformly distributed from 90 to 100
270  degrees. B{This will be set to 0 if not given.}
271  - each forward or backward move is set at 1 (set_distance).
272  B{This setting is mandatory.}
273  - random distance of each move can be set using 'random_
274  distance', following the same logic as 'random_angle'. B{This
275  will be set to 0 if not given.}
276  - turtle is set to head towards 0 degrees (east or to the
277  right of the screen). North (toward top), west (towards left),
278  and south (towards bottom) are 90, 180, 270 degrees
279  respectively. B{This will be set to 0 if not given.}
280  - default pen colour can be set using TK colour names as
281  'set_colour'. B{This will be set to black if not given.} Other
282  colours can be set and any un-used symbols (other than 'F', 'B',
283  'R', 'L', 'H', 'U', and 'D') can be used to set pen colours
284  (please see http://wiki.tcl.tk/37701 for available colours).
285  - canvas background colour can be set using background_colour.
286  B{This will be set to ivory if not given.}
287  - 'F', 'B', 'R', 'L', 'H', 'U', and 'D' represents the Turtle
288  commands of forward, backward, right turn, left turn, home,
289  pen up, and pen down respectively. Home is defined as the start
290  coordinate.
291  - '[' pushes the current state (position and heading) of the
292  Turtle into the stack.
293  - ']' pops (in a last in first out manner) and sets the Turtle
